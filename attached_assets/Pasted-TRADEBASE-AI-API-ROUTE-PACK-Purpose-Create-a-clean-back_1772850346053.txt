TRADEBASE — AI API ROUTE PACK
=============================

Purpose:
Create a clean backend structure for all AI-powered features inside TradeBase.

All AI routes must:
- run server-side only
- use OPENAI_API_KEY from environment variables
- return strict JSON
- never expose the API key to the client
- validate outputs before using them
- fail safely if AI output is invalid

============================================================
BASE SETUP
============================================================

Environment Variables Required

OPENAI_API_KEY
NEXT_PUBLIC_SUPABASE_URL
NEXT_PUBLIC_SUPABASE_ANON_KEY
SUPABASE_SERVICE_ROLE_KEY (optional if needed server-side)

Folder Structure

/api/ai/quote-builder
/api/ai/talk-to-work
/api/ai/permit-assistant
/api/ai/follow-up
/api/ai/job-brain
/api/ai/job-note
/api/ai/daily-ops
/api/ai/client-intelligence

============================================================
1. /api/ai/quote-builder
============================================================

Method:
POST

Purpose:
Generate a structured quote draft using service presets and customer info.

Request Body Example:

{
  "org_id": "uuid",
  "business_name": "Central Maine Radon",
  "service_presets": [
    {
      "id": "preset-1",
      "name": "Radon Mitigation Install",
      "description": "Standard basement radon mitigation system",
      "default_price": 1200
    }
  ],
  "client_name": "Mike Johnson",
  "client_address": "45 Elm St Augusta ME",
  "user_request": "Create quote for radon mitigation install"
}

Response Example:

{
  "job_title": "Radon Mitigation Install",
  "line_items": [
    {
      "preset_id": "preset-1",
      "description": "Radon Mitigation Install",
      "qty": 1,
      "unit_price": 1200,
      "needs_manual_pricing": false
    }
  ],
  "scope_description": "Standard basement radon mitigation system installation.",
  "notes": null,
  "needs_review": false
}

============================================================
IMPLEMENTATION REQUIREMENTS
============================================================

1. All endpoints must call OpenAI server-side only.
2. Use strict JSON output for every feature.
3. Validate responses before returning them to the frontend.
4. If AI output is invalid:
   - return an error state
   - do not save to database
5. Add preview / review screen before creating records.
6. Log AI runs if AI history is enabled.
