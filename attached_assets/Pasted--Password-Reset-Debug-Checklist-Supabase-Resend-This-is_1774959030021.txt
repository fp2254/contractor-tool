🚨 Password Reset Debug Checklist (Supabase + Resend)

This is a step-by-step checklist to fix "cannot connect to host" issues when resetting passwords.

---

# 🔍 Step 1 — Check Supabase Project Status
- Go to Supabase dashboard
- Open your project
- Confirm it is NOT paused

👉 Free tier projects can sleep and break auth requests

---

# 🔍 Step 2 — Verify Environment Variables
Make sure these are set correctly:

NEXT_PUBLIC_SUPABASE_URL=
NEXT_PUBLIC_SUPABASE_ANON_KEY=

Checklist:
- No extra spaces
- No incorrect quotes
- URL should look like:
  https://xxxxx.supabase.co

---

# 🔍 Step 3 — Test Reset Function
Use this exact function:

await supabase.auth.resetPasswordForEmail(email, {
  redirectTo: 'https://tradebase.contractors/reset-password'
})

---

# 🔍 Step 4 — Configure Redirect URLs (VERY IMPORTANT)
In Supabase:

Authentication → URL Configuration

Add:

https://tradebase.contractors/reset-password
https://tradebase.contractors/auth/callback

👉 If this is wrong, reset will fail

---

# 🔍 Step 5 — Check Resend Setup
- Domain verified
- SPF and DKIM configured
- Not hitting sending limits

⚠️ Note:
If error is "cannot connect to host" → this is likely NOT Resend

---

# 🔍 Step 6 — Check Network Requests
Open Dev Tools → Network tab

Look for:
POST /auth/v1/recover

Status codes:
- 200 → Success
- 400 → Redirect issue
- 401 → Wrong API key
- 404 → Wrong URL
- Failed / 0 → Network/host issue

---

# 🔥 Step 7 — Verify Supabase Client Setup
Make sure you're importing correctly:

import { createClient } from '@supabase/supabase-js'

---

# ⚠️ Step 8 — Replit Issues
If using Replit:
- Restart project
- Clear cache
- Redeploy

---

# 💥 Most Likely Causes
1. Supabase project sleeping
2. Redirect URL mismatch
3. Incorrect environment variables

---

# 🧠 Fast Fix Order
1. Confirm Supabase is active
2. Re-copy env variables
3. Add redirect URLs
4. Refresh app
5. Check network tab

---

If still broken:
Send error message + network screenshot + reset function for deeper debuggin
