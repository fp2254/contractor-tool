1) X-Hit-Express IS PRESENT on the main POST/PUT /api/invoices routes.

3) PostgREST errors ONLY occur when Supabase is being used for DB writes.
   → There is still at least ONE supabaseAdmin.from() call in the invoice save flow.

This is NOT a frontend validation problem.
This is NOT a UUID regex problem.
This is NOT Safari cache.

This IS an architectural violation.

- The UI allows optional inputs (customer, address, job, photos, undo, activity log, offline sync).
- One of those paths is still:
  supabaseAdmin.from(...).insert/update(...)
- When ANY optional UUID field is empty or contains text, Supabase throws PostgREST:
  "string did not match expected pattern"

   Frontend → apiFetch → Express → pgPool

   - Validate UUID format
